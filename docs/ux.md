# UX

주요 화면, 랜덤화 정책, localStorage 저장, 백업/복원 흐름.

## 1. 주요 화면

### 1.1 홈

- 과목 카드 그리드
- 마지막 풀이 세션 이어보기 버튼
- 북마크/오답 기록 보기 진입점
- 백업·복원 진입점

### 1.2 과목/문제 선택

- 과목 선택 (홈에서 진입 시 자동 선택)
- 출처 종류 탭: 기출 / 교재 / 강의
- 출처별 체크리스트 (다중 선택 가능)
  - 기출 탭: 연도별 토글 (2017/2018/2019)
  - 교재 탭: 챕터별 체크박스
  - 강의 탭: 강의별 체크박스
- 추가 필터:
  - 북마크된 문제만
  - 이전에 틀린 문제만
- 옵션:
  - 출제 문항 수 제한 (전체 / 무작위 N문항)
  - 시드 입력(고급) → 동일 세트 재현
- 선택된 문제 수 표시
- 풀이 시작 버튼

### 1.3 문제 풀이

- 상단: 진행 표시(현재/전체), 북마크 토글, 종료
- 공통 지문이 있으면 문제 상단에 펼침 형태로 표시
- 문제 본문 (마크다운/수식)
- 첨부 이미지
- 선택지 목록 (라디오/체크박스, 순서 무작위)
- 정답 확인 버튼
- 정답 확인 후:
  - 정답·오답 표시 (선택지 색상)
  - 해설 영역 표시 (마크다운/수식/코드 하이라이트)
  - 자동 북마크 토글 (오답 시, 사용자 설정)
- 이전 / 다음 문제 버튼

### 1.4 결과

- 총 문항, 정답 수, 정답률
- 문제별 정·오답 요약 (클릭 시 상세보기)
- 틀린 문제만 다시 풀기
- 새 세션 시작 / 홈으로

### 1.5 북마크

- 북마크한 문제 목록
- 과목/출처 필터
- 북마크 문제만 다시 풀기
- 북마크 해제

### 1.6 백업/복원

서버와 DB를 사용하지 않으므로 사용자별 북마크와 오답 기록은 자동 동기화되지 않는다. 사용자가 직접 JSON 파일로 내보내고 다른 기기에서 가져온다.

- 내보내기: 현재 `localStorage` 데이터를 JSON 파일로 다운로드
- 가져오기: 사용자가 선택한 JSON 파일을 검증 후 병합 또는 덮어쓰기
- 가져오기 전 충돌 시 병합/덮어쓰기 선택 UI 제공
- 잘못된 JSON, 버전 불일치, 필수 필드 누락에 대한 오류 메시지 제공

## 2. 랜덤화 정책

- 문제 순서: 풀이 세션 시작 시 Fisher-Yates 셔플로 섞는다.
- 선택지 순서: 문제 표시 시 셔플하되, 정답 판정은 표시 순서가 아니라 `choice.id` 기준으로 한다.
- 시드: 기본은 `Math.random()` 기반. 고급 옵션으로 사용자가 시드 입력 시 재현 가능한 세트를 만든다 (URL `?seed=...` 공유).
- 같은 세션 내에서는 순서를 유지한다. 새로고침 시 진행 중 세션을 복구할지(`pt.lastSession`) 새 세션으로 시작할지 선택 UI를 보여준다.

## 3. localStorage 저장 항목

키 prefix는 `pt.`로 통일한다.

```text
pt.bookmarks       # string[] (복합 키 배열)
pt.wrongAnswers    # Record<복합키, { wrongCount, lastWrongAt }>
pt.history         # Record<복합키, { lastResult, attempts, lastAt }>
pt.settings        # 사용자 설정 (자동 북마크 등)
pt.lastSession     # 진행 중 세션 스냅샷
```

`pt.bookmarks` 예시:

```json
[
  "operating-systems:past-exams-2019:e19-01",
  "algorithms:workbook:b03-07"
]
```

`pt.wrongAnswers` 예시:

```json
{
  "operating-systems:past-exams-2019:e19-01": {
    "wrongCount": 2,
    "lastWrongAt": "2026-05-06T12:30:00.000Z"
  }
}
```

초기 버전에서는 북마크와 오답 기록을 함께 구현한다. `history`는 2차로 미룬다.

복합 키 형식과 식별자 규칙은 [data-schema.md §4](./data-schema.md) 참고.

## 4. 백업: 내보내기

내보내기 버튼을 누르면 현재 브라우저의 사용자 데이터를 하나의 JSON 파일로 생성한다.

파일명 예시:

```text
passture-user-data-2026-05-06.json
```

내보내기 데이터 예시:

```json
{
  "app": "passture",
  "version": 1,
  "exportedAt": "2026-05-06T12:50:00.000Z",
  "bookmarks": [
    "operating-systems:past-exams-2019:e19-01"
  ],
  "wrongAnswers": {
    "algorithms:workbook:b03-07": {
      "wrongCount": 2,
      "lastWrongAt": "2026-05-06T12:40:00.000Z"
    }
  }
}
```

## 5. 백업: 가져오기

가져오기는 사용자가 선택한 JSON 파일을 브라우저에서 읽고 검증한 뒤 `localStorage`에 반영한다.

방식:

- 병합: 기존 데이터를 유지하면서 가져온 데이터를 추가한다.
- 덮어쓰기: 기존 데이터를 삭제하고 가져온 데이터로 교체한다.

병합 규칙:

- 북마크는 중복 제거 후 합친다.
- 오답 기록은 같은 키가 있으면 `wrongCount`는 큰 값, `lastWrongAt`은 더 최근 값을 사용한다.
- 알 수 없는 문제 키는 삭제하지 않고 보존한다 (이후 데이터 추가/복구 가능성).

## 6. 백업: 검증 규칙

가져오기 파일은 최소한 다음 조건을 만족해야 한다.

- 최상위 `app` 값이 `"passture"`
- `version`이 지원 가능한 버전
- `bookmarks`가 문자열 배열
- `wrongAnswers`가 객체
- 문제 키는 `{subjectId}:{sourceId}:{questionId}` 형식

검증에 실패하면 `localStorage`를 변경하지 않고 오류를 표시한다.
