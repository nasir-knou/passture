# UX

현재 구현된 주요 화면, 랜덤화 정책, 저장소 사용, 데이터 관리 흐름.

## 1. 주요 화면

### 1.1 홈

- 과목 카드 그리드
- 학기 필터: `전체 / 1학기 / 2학기`
- 과목 카드에는 `1학기` 또는 `2학기` 배지를 표시한다.
- 상단바: 문제 선택, 모의 시험, 학습 기록, 데이터 관리
- 과목 카드의 선택 버튼으로 해당 과목 선택 화면 진입

### 1.2 과목/문제 선택

- 과목 선택 (홈에서 진입 시 자동 선택)
- 학기 필터: `전체 / 1학기 / 2학기`
- 선택 화면 안에서 과목 select로 다른 과목 전환. 과목 옵션은 선택한 학기 필터에 맞춰 표시하고, `1학기 · 운영체제`처럼 학기와 과목명을 함께 표시한다.
- 홈에서 특정 과목으로 진입하면 해당 과목의 학기를 필터 기본값으로 사용한다.
- 선택된 과목의 출처 체크리스트 (다중 선택 가능)
  - 큰 분류는 `기출 / 교재 / 강의`만 사용한다.
  - 교재 분류 안에 `기본서 문제`, `워크북 문제`를 둔다.
  - 강의 분류 안에 `연습문제`, `특강 문제`를 둔다.
  - 출처명 옆에는 빌드된 문제 수를 `2019 기말 (25문제)` 형식으로 표시한다.
- 문제 범위:
  - 전체
  - 북마크만
  - 오답만
- 옵션:
  - 문제 순서: 기본 / 순서 무작위
  - 선지 순서: 기본 / 순서 무작위
- 풀이 시작 버튼

### 1.3 문제 풀이

- 상단: 답안 체크 기준 진행률, 현재 문제/전체 문제 표시, 북마크 토글
- 문제 번호 지도:
  - 현재 문제
  - 답안 체크 안 함
  - 답안 체크함
  - 북마크 표시
  - 번호 클릭으로 해당 문제 이동
- 공통 지문이 있으면 문제 상단에 표시
- 공통 지문의 데이터 식별자(`passages.id`)나 내부 라벨은 화면에 표시하지 않는다.
- 텍스트 지문은 수식을 렌더링하고, 이미지 지문은 crop 이미지를 표시하며, diagram 지문은 구조화 데이터에서 도표를 렌더링한다.
- 문제 본문
- 첨부 이미지
- 선택지 목록 (라디오/체크박스, 순서 무작위)
- 이전 / 정답 확인 / 다음 버튼
- 정답 확인 후:
  - 정답·오답 표시 (선택지 색상)
  - 해설 영역 표시
- 모든 문제에 답안을 체크하면 하단 `채점하기` 버튼 활성화

### 1.4 결과

- 출처 세트 시각화
- 총 문항, 응답, 정답, 정답률 카드
- 문제별 정·오답 지도
- 번호 클릭 시 해당 문제, 선택지, 정답/오답, 해설 표시
- 풀이로 돌아가기 / 새 세션 시작

### 1.5 학습 기록

- URL: `#/history`
- 과거 `#/bookmarks` URL은 `#/history`로 이동한다.
- 설명: 북마크한 문제와 오답 문제를 모아두는 공간
- 요약: 전체, 북마크, 오답 문제 수
- 필터:
  - 기록 유형: 전체 / 북마크 / 오답 문제
  - 과목: 전체 과목 / 개별 과목
  - 출처: 전체 출처 / 개별 출처
- 모두 펼치기 / 모두 접기
- 문제 카드는 기본 접힘 상태:
  - 접힘: 문제 ID, 과목/출처, 북마크/오답 배지, 문제 본문
  - 펼침: 지문, 선택지, 북마크 해제 버튼

### 1.6 데이터 관리

Passture는 별도의 서버/데이터베이스를 사용하지 않고 데이터를 로컬 브라우저에만 저장한다. 사용자별 북마크와 오답 기록은 자동 동기화되지 않으므로 사용자가 직접 JSON 파일로 내보내고 다른 기기에서 가져온다.

- 내보내기: 현재 `localStorage` 데이터를 JSON 파일로 다운로드
- 가져오기: 사용자가 선택한 JSON 파일을 검증 후 병합 또는 덮어쓰기
- 가져오기 전 충돌 시 병합/덮어쓰기 선택 UI 제공
- 잘못된 JSON, 버전 불일치, 필수 필드 누락에 대한 오류 메시지 제공
- 현재 풀이 초기화: `sessionStorage`의 출처 선택, 범위/순서 옵션, 현재 풀이 세션 삭제
- 학습 기록 초기화: `localStorage`의 북마크와 오답 기록 삭제. 풀이 세션도 함께 삭제

### 1.7 모의 시험

- 상단바 메뉴와 `#/mock-exam`, `#/mock-exam/test`, `#/mock-exam/result` 라우트가 존재한다.
- 저장된 문제를 유사 기말 시험 환경에서 타이머와 함께 푸는 방식이다.
- 문제는 기존 YAML/JSON으로 빌드된 저장 문제 풀에서 구성한다.
- 학기 필터: `전체 / 1학기 / 2학기`
- 접속 기기의 로컬 시간 기준으로 2월~7월은 1학기, 8월~1월은 2학기를 기본 필터로 선택한다.
- 과목은 1~3개까지 선택할 수 있고, 시험 시간은 과목 수 × 25분으로 계산한다.
- 과목별로 하나의 출처만 선택할 수 있다. 여러 과목을 선택해도 각 과목은 독립된 단일 출처에서 문제를 가져온다.
- 기출 출처는 25문항이면 그대로 사용하고, 35문항이면 무작위로 25문항을 추출한다.
- 교재/워크북/강의/특강 출처는 문제 ID의 첫 숫자 그룹을 기준으로 전체 그룹 범위를 파악한 뒤 25문항을 균등 분포로 추출한다. 예: 최대 그룹이 `07`이면 각 그룹에서 3~4문항, 최대 그룹이 `11`이면 2~3문항, 최대 그룹이 `15`이면 1~2문항을 뽑는다.
- 문제 수가 부족한 그룹이 있으면 가능한 만큼 뽑고, 부족분은 아직 선택되지 않은 전체 후보에서 무작위로 채워 최종 25문항을 맞춘다.
- 풀이 중에는 진행 시간, 남은 시간, 문항 이동, 임시 답안 상태, 북마크를 보여준다.
- 제출 또는 시간 종료 후 시험형 결과 리포트를 표시한다.

### 1.8 스크롤 정책

- 라우트 이동 시 기본적으로 브라우저 스크롤 위치를 유지한다.
- 문제 선택 화면(`/select`)에 새로 진입할 때만 페이지 최상단으로 이동한다.

## 2. 랜덤화 정책

- 문제 순서: 사용자가 `순서 무작위`를 선택하면 풀이 세션 생성 시 Fisher-Yates 셔플로 섞는다. 기본값은 데이터 순서다.
- 선택지 순서: 사용자가 `순서 무작위`를 선택하면 세션 생성 시 섞는다. 정답 판정은 표시 순서가 아니라 `choice.id` 기준으로 한다.
- 시드: 아직 지원하지 않는다.
- 같은 세션 내에서는 순서를 유지한다. 세션은 `sessionStorage`에 저장되며, 같은 출처/범위/순서 옵션이면 새로고침 후에도 이어진다.

## 3. localStorage 저장 항목

키 prefix는 `pt.`로 통일한다. 영구 사용자 데이터는 `localStorage`, 현재 풀이 세션과 선택 옵션은 `sessionStorage`에 저장한다.

```text
localStorage:
pt.bookmarks       # string[] (복합 키 배열)
pt.wrongAnswers    # Record<복합키, { wrongCount, lastWrongAt }>

sessionStorage:
pt.selectedSources # 현재 선택한 출처 배열
pt.practiceScope   # all | bookmarked | wrong
pt.practiceOptions # questionOrder / choiceOrder
pt.currentSession  # 진행 중 풀이 세션 스냅샷
pt.mockExamConfig  # 모의시험 설정
pt.mockExamSession # 진행 중 모의시험 세션 스냅샷
```

일부 풀이 화면 UI 상태는 기존 호환 키인 `passture.questionMapExpanded`, `passture.quiz.scrollY`를 사용한다.

`pt.bookmarks` 예시:

```json
["operating-systems:past-exams-2019:e19-01", "algorithms:workbook:b03-07"]
```

`pt.wrongAnswers` 예시:

```json
{
  "operating-systems:past-exams-2019:e19-01": {
    "wrongCount": 2,
    "lastWrongAt": "2026-05-06T12:30:00.000Z"
  }
}
```

사용자 설정, 자동 북마크는 아직 구현하지 않았다.

복합 키 형식과 식별자 규칙은 [data-schema.md §4](./data-schema.md) 참고.

## 4. 백업: 내보내기

내보내기 버튼을 누르면 현재 브라우저의 사용자 데이터를 하나의 JSON 파일로 생성한다.

파일명 예시:

```text
passture-user-data-20260506-010203.json
```

내보내기 데이터 예시:

```json
{
  "app": "passture",
  "version": 1,
  "exportedAt": "2026-05-06T12:50:00.000Z",
  "bookmarks": ["operating-systems:past-exams-2019:e19-01"],
  "wrongAnswers": {
    "algorithms:workbook:b03-07": {
      "wrongCount": 2,
      "lastWrongAt": "2026-05-06T12:40:00.000Z"
    }
  }
}
```

## 5. 백업: 가져오기

가져오기는 사용자가 선택한 JSON 파일을 브라우저에서 읽고 검증한 뒤 `localStorage`에 반영한다.

방식:

- 병합: 기존 데이터를 유지하면서 가져온 데이터를 추가한다.
- 덮어쓰기: 기존 데이터를 삭제하고 가져온 데이터로 교체한다.

병합 규칙:

- 북마크는 중복 제거 후 합친다.
- 오답 기록은 같은 키가 있으면 `wrongCount`는 큰 값, `lastWrongAt`은 더 최근 값을 사용한다.
- 알 수 없는 문제 키는 삭제하지 않고 보존한다 (이후 데이터 추가/복구 가능성).

## 6. 백업: 검증 규칙

가져오기 파일은 최소한 다음 조건을 만족해야 한다.

- 최상위 `app` 값이 `"passture"`
- `version`이 지원 가능한 버전
- `bookmarks`가 문자열 배열
- `wrongAnswers`가 객체
- 문제 키는 `{subjectId}:{sourceId}:{questionId}` 형식

검증에 실패하면 `localStorage`를 변경하지 않고 오류를 표시한다.
