# Roadmap

구현 단계, 검증 항목, 우선순위.

## 1. 구현 단계

1. 프로젝트 구조 생성 (Vite + TS 스캐폴딩, `tsconfig`, `vite.config.ts`, `base` 설정)
2. 타입 정의 (`Catalog`, `QuestionFile`, `Question`, `Passage`, `Choice`, `UserData`)
3. 데이터 빌드 스크립트 작성 (`scripts/build-data.ts` + 스키마 검증)
4. 샘플 YAML 작성 (운영체제 2019 기말 5문항 + 공통지문 1개)
5. 데이터 로더 구현 (`catalog.json` → 출처 JSON 지연 로딩)
6. 라우터와 페이지 골격 (홈 / 선택 / 모의 시험 / 풀이 / 결과 / 학습 기록 / 데이터 관리)
7. 문제 선택 화면 구현
8. 풀이 엔진 구현 (셔플, 진행 상태, 채점)
9. 정답·해설 표시
10. 북마크 저장/해제
11. 오답 기록 저장 + 오답만 풀기 모드
12. 결과 화면
13. 학습 기록 화면 (북마크/오답 문제 필터, 과목/출처 필터, 접이식 문제 카드)
14. 데이터 관리 화면 (내보내기 → 가져오기 검증/병합, 풀이/학습 기록 초기화)
15. GitHub Actions 배포 워크플로우
16. 콘텐츠 입력: 운영체제 기출 3년치 → 워크북 → 강의 순
17. 공식 정오표/정답표 기반 정답 검증
18. 모바일 반응형, 접근성 점검

## 2. 검증 항목

### 2.1 사이트 동작

- GitHub Pages 커스텀 도메인 배포(`https://passture.logonme.click/`)에서 모든 자원 경로가 정상이다.
- `catalog.yaml`에 과목/출처를 추가하고 빌드만 다시 돌리면 UI에 자동 반영된다.
- `catalog.yaml`의 모든 과목은 `semester: 1 | 2`를 가지며, 홈에서 전체/1학기/2학기 필터가 정상 동작한다.
- 빌드된 카탈로그에는 출처별 문제 수가 포함되고, 홈/선택 화면에 `(N문제)`로 표시된다.
- 여러 출처를 동시에 선택해 한 세션에서 풀 수 있다.
- 사용자가 `순서 무작위`를 선택하면 문제 순서가 세션 생성 시 섞이고, 기본값은 데이터 순서다.
- 선택지 순서가 섞여도 정답 판정이 정확하다 (`choice.id` 기반).
- 복수 정답 문제(`multi-answer`)가 정확히 판정된다.
- O/X 문제가 정확히 판정된다.
- 공통 지문이 여러 문제에 정상 표시된다.
- 이미지가 상대 경로(`base` 포함)로 정상 표시된다.
- 북마크가 브라우저 재접속 후에도 유지된다.
- 틀린 문제가 오답 기록에 저장된다.
- `#/history`에서 북마크/오답 문제를 기록 유형, 과목, 출처 기준으로 필터링할 수 있다.
- 학습 기록 문제 카드는 기본 접힘 상태이며, 모두 펼치기/모두 접기 버튼이 동작한다.
- 북마크/오답 기록을 JSON 파일로 내보낼 수 있다.
- 내보낸 JSON 파일을 다시 가져오면 데이터가 복원된다.
- 가져오기 병합 시 중복 북마크가 생기지 않는다.
- 잘못된 가져오기 파일은 기존 데이터를 변경하지 않는다.

### 2.2 데이터 빌드

- ID 중복(같은 파일 내) 발견 시 빌드 실패.
- `passageRefs`의 미존재 ID 발견 시 빌드 실패.
- `answers`의 항목이 `choices.id`에 없으면 빌드 실패.
- `images.path` 파일이 저장소에 없으면 빌드 실패.
- `diagram` 타입별 필수 필드와 노드/간선 참조가 올바르지 않으면 빌드 실패.
- 식별자 형식 위반(예: 기출인데 `e` 접두 누락) 시 빌드 실패.
- 교재/워크북/강의/특강 문제는 ID의 첫 숫자 그룹을 모의시험 분산 기준으로 사용할 수 있도록 출처 안에서 일관되게 작성한다.

### 2.3 정답 검증

- 콘텐츠 입력 단계(M6~M9)에서는 공식 정오표/정답표 확인 전이라도 문제 판독과 직접 풀이로 임시 정답을 기재한다.
- 임시 정답에는 풀이 근거를 `explanation`에 남겨 이후 대조가 가능하도록 한다.
- M10에서 공식 정오표/정답표로 모든 기출 정답을 대조하고, 불일치 문항은 `answers`와 `explanation`을 수정한다.
- 정답 검증 완료 여부와 수정 문항은 작업 기록에 남긴다.

## 3. 우선순위

### 3.1 완료된 범위

- Vite+TS 스캐폴딩, base 설정, GitHub Pages 배포
- catalog.yaml + 운영체제/알고리즘/인공지능/Java 출처 등록
- 데이터 빌드 스크립트(YAML→JSON, 기본 검증)
- 과목/출처 선택 화면 (과목 select + 출처 체크리스트)
- 문제 풀이 화면 (셔플, 문제 지도, 답안 체크 진행률, 개별 정답 확인, 전체 채점, 해설)
- 결과 화면 (출처 세트, 정답/오답 문제 지도, 선택 문항 해설)
- 북마크 저장
- 오답 기록 저장, 북마크만/오답만 풀기
- 학습 기록 화면 (북마크/오답, 기록 유형/과목/출처 필터, 접이식 카드)
- 북마크/오답 내보내기·가져오기
- 모의 시험 설정/풀이/결과 화면 (1~3과목, 과목당 단일 출처, 과목당 25분, 25문항 구성, 제출/시간 종료 처리)
- KaTeX 수식 렌더링
- 구조화 다이어그램 렌더링 (`resource-allocation-graph`, `memory-free-list`, `data-table`, `clock-page-replacement`)

### 3.2 다음 개선 후보

- 모의 시험 UX 개선: ID 그룹 기반 균등 추출의 설명과 예외 안내를 보강한다.
- 학기 필터 확장: 과목 수가 늘어나면 학습 기록과 모의 시험에도 1학기/2학기 필터를 추가한다.
- 연도/ID 그룹/강의 단위 필터
- 세션 복구 UX 개선 (현재는 같은 조건이면 `sessionStorage` 세션을 자동 재사용)
- 이미지 빌드 시 경고/에러 분리
- 모바일 반응형과 접근성 점검 보강

### 3.3 3차 개선

- 검색 (문제 본문/태그)
- 태그 필터
- 시드 기반 세트 공유 URL
- 공식 정오표/정답표 기반 정답 대조
- 마크다운/코드 하이라이트 렌더링
- 다크 모드
- 문제 작성 보조 도구 (브라우저 내 YAML 입력 폼)
- 이미지 빌드 최적화 (WebP)
- 학습 통계 (태그별 정답률)
