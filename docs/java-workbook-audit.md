# Java Workbook Audit

Java 워크북(`data/subjects/java-programming/workbook.yaml`)을 원본 이미지(`origin/Java 워크북`)와 10문항 단위로 대조한 기록이다.

## 기준

- 원본 문제의 prompt, 선택지, 정답이 YAML에 같은 의미로 들어가 있어야 한다.
- 주관식, 서술형, 프로그래밍 작성형, 원문 의미를 선택형으로 안정적으로 보존하기 어려운 문항은 제외한다.
- 원본 해설이 짧은 경우에도 YAML 해설은 정답 근거, 각 오답의 이유, 핵심 개념 요약을 포함해야 한다.
- Java 워크북의 현재 포함 문항은 코드와 간단한 표기만 필요하여 이미지 crop 없이 prompt/choice 텍스트로 구조화한다.
- 검증 단위는 YAML 배열 순서 기준 10문항이다.

## 발견 사항

- `b03-07`은 원본 질문이 "잘못된 것이 있으면 모두 고르시오"이고 원본 정답은 "모두 올바른 설명이다"이다. 현재 앱 선택형으로는 "없음"을 답으로 표현할 수 없어 문제 의미가 바뀌므로 YAML에서 제외하고 `issue.md`에 추가했다.
- Java 워크북 포함 문항은 모두 객관식 또는 복수정답 선택형이며, 도표 crop이 필요한 문항은 없다.
