# 방통대 기말고사 대비 사이트 작업문서

## 서비스명

**PASSture** — pasture(목초지) + pass(합격)의 합성. "양치기" 학습법에서 컨셉을 가져왔다.

> 양치기: 공부 없이 문제풀이 반복만으로 시험을 대비하는 한국식 학습 속어. 양떼를 몰듯 문제를 갈아넣는다는 어감.

PASSture는 그 양떼가 풀을 뜯는 목초지를 뜻한다. 사용자가 문제를 떼로 몰아 풀고, 그 과정에서 합격(pass)에 도달하는 공간이라는 의미.

## 1. 목표

방통대 기말고사 대비 문제풀이 사이트를 GitHub Pages 기반 정적 사이트로 제작한다.

- 서버·DB 없이 정적 파일만으로 운영 (운영비 0)
- 과목별·출처별 문제 선택, 문제·선택지 순서 셔플로 익숙함 제거
- 오답·즐겨찾기 북마크와 재학습
- 새 과목/출처를 코드 수정 없이 추가할 수 있는 확장 구조
- 사람이 작성하기 쉬운 데이터 포맷 + 빌드 시 검증·최적화

## 2. 결정 사항 요약

- 프론트엔드: **Vite + TypeScript** SPA, 자체 미니 **해시 라우터**(`#/select`, `#/mock-exam`, `#/mock-exam/test`, `#/mock-exam/result`, `#/quiz`, `#/result`, `#/history`, `#/backup`)
- 패키지 매니저/런타임: **pnpm** + **Node 22 LTS** (`.nvmrc`)
- 코드 스타일: **Prettier만** (eslint 보류)
- 테스트: **vitest** — 데이터 빌드, 라우터, 채점, 세션, 저장소, 백업 단위 테스트
- 데이터: **YAML 저작 → JSON 빌드** (런타임은 JSON만 fetch)
- 데이터 산출 경로: **`public/data/`** 직접 출력 + gitignore (Vite가 dist로 자동 복사)
- 카탈로그: `catalog.yaml`(빌드 후 `catalog.json`)에서 과목/출처 관리
- 문제 파일: 출처별 단일 YAML/JSON
- 공통 지문: `passages` 분리 + `passageRefs` 배열 참조
- 정답: 단일/복수/OX 모두 `answers` 배열로 통일
- 식별자 컨벤션
  - 기출: `e{yy}-{nn}` (예: `e19-01`)
  - 교재: `t{chapter}-{nn}` (예: `t03-07`)
  - 워크북: `b{chapter}-{nn}` (예: `b03-07`)
  - 강의: `l{lecture}-{nn}` (예: `l07-08`)
  - 공통지문: `g` 접두 (예: `g19-code-01`)
- 교재/워크북/강의/특강 출처의 모의시험 분산 추출은 문제 ID의 첫 숫자 그룹을 기준으로 한다. 예: `b03-07`, `l11-04`. `chapter:NN` 같은 태그는 선택 메타데이터이며 빌드 필수 조건이 아니다.
- 사용자 데이터: 영구 데이터는 `localStorage`, 풀이 세션/선택 옵션은 `sessionStorage` (`pt.` 키 prefix)
- 데이터 관리: JSON 파일 내보내기·가져오기 (병합/덮어쓰기), 풀이/학습 기록 초기화
- 문제 선택 UI: 홈 과목 카드 → 선택 화면, 전체/1학기/2학기 필터, 선택 화면 안에서 과목 전환 + 출처 체크리스트 + 과목별 최근 학습 상태
- 문제 풀이 UI: 답안 체크 기준 진행률, 문제 번호 지도, 개별 정답 확인과 다음 이동 분리, 전체 채점 버튼
- 학습 기록 UI: `#/history`에서 북마크/오답 문제를 기록 유형·과목·출처로 필터링하고 접이식 문제 카드로 확인
- 데이터 관리 UI: `#/backup`에서 사용자 데이터 내보내기/가져오기와 현재 풀이/학습 기록 초기화
- 렌더링: KaTeX 수식, 이미지 지문, 코드 지문, 구조화 다이어그램 지문을 지원한다.
- 공통 지문 정책: 도표/그래프/표는 텍스트로 해석해 저장하지 않는다. 브라우저에서 구조화 렌더링할 수 있는 이미지는 `passages[].type: diagram`과 `diagram` 필드로 코드화한다. diagram으로 표현하기 어려운 원본 시각 정보가 문제 조건인 경우에만 `passages[].image.path`로 crop 이미지 경로를 저장한다. 현재 diagram은 자원할당 그래프, 일반 그래프, Java UI 창, 빈 공간 리스트, 표, 클럭 페이지 교체 원형 큐를 지원한다. `passages.id`는 화면에 노출하지 않는다.
- 데이터 무결성 검증은 빌드 단계 강제, GitHub Actions로 PR마다 실행
- 배포: GitHub Pages, **커스텀 도메인 `passture.logonme.click`**, `vite.config.ts`의 `base = "/"`
- 계정/레포: 신규 계정 **`nasir-knou`** + 레포 `nasir-knou/passture` (이메일·gh 인증·SSH 키 분리)
- DNS: 도메인 등록업체에서 `passture` 호스트 → `nasir-knou.github.io.` CNAME, `nasir-knou` 계정에서 `logonme.click` TXT verify
- 라이선스: 코드 **MIT**. 데이터(기출/교재 인용)는 README·푸터에 비영리 학습 목적 + 한국방송통신대학교 및 한국방송통신대학교 출판문화원 권리 표기 + 권리자 요청 시 삭제 표기

## 8. 이어서 작업할 때 참고

- 원격 공개 저장소 기준점은 `5c462b7 feat(ui): refine subject selection flow`이며, 로컬 `main`에는 이후 UX 개선 커밋이 추가되어 있다.
- 로컬에 M6 입력 자료로 운영체제 기말 PDF 3개가 있음: `17-운영체제-3학년-3교시.pdf`, `18-운영체제-3학년-3교시.pdf`, `19-운영체제-3학년-3교시.pdf`.
- PDF 원문은 공개 저장소에 올리지 않도록 `.gitignore`에 `*.pdf`를 추가함.
- 2019 PDF는 `pypdf` 텍스트 추출 시 폰트 인코딩 문제로 본문이 깨짐. `PyMuPDF`로 페이지 이미지는 렌더링 가능했으므로 M6는 이미지 판독/OCR 또는 수동 입력으로 진행해야 함.
- 콘텐츠 입력 단계에서는 공식 정오표/정답표 확인 전이라도 `answers`를 비워두지 않는다. 문제 이미지를 판독하고 직접 풀이한 임시 정답과 해설을 기재한 뒤, 공식 정답표가 있으면 세트 입력 완료 후 정답 문자열을 대조한다.
- 해설은 각 선택지별 정답/오답 판단 이유와 핵심 개념 4~5줄 요약을 함께 작성한다. 오답 선택지는 해당 선택지가 가리키는 개념을 설명하고, 왜 문제 조건이나 질문과 맞지 않는지 적는다.
- 약어가 선택지나 해설의 핵심 판단에 등장하면 첫 설명에서 풀폼을 함께 적는다. 예: RTOS(Real-Time Operating System), NUMA(Non-Uniform Memory Access).
- 표/그래프/도식 이미지는 도표화/코드화가 가능하면 diagram으로 작성한다. diagram으로 재현하기 어렵거나 원본의 세부 시각 형태 자체가 문제 조건인 불가피한 경우에만 crop 이미지를 사용한다.
- 상단바 기본 메뉴는 문제 선택, 모의 시험, 학습 기록, 데이터 관리다.
- 모의 시험은 `#/mock-exam` 설정 화면, `#/mock-exam/test` 시험 화면, `#/mock-exam/result` 결과 화면으로 구성된다. 최대 3과목, 과목당 25분, 과목별 단일 출처, 전체/무작위 25문항 구성을 지원한다.
- 선택 화면은 홈의 과목 카드에서 `#/select?subject={subjectId}`로 진입하며, 같은 화면의 과목 select로 다른 과목 출처를 즉시 전환한다.
- 풀이 화면의 진행률은 답안을 체크한 문제 수 기준이다. 현재 문제 위치와 별개로 전체 답안 체크 진행 정도를 표시한다.
- 정답 확인은 개별 문제 채점이며, 최종 결과는 모든 문제 답안을 체크한 뒤 하단 `채점하기` 버튼으로 이동한다.
- 결과 화면은 현재 세션의 출처 세트를 보여주고, 문제 번호를 클릭하면 해당 문항의 문제/선택지/해설을 확인한다.
- 학습 기록은 `#/history`가 정식 URL이다. 기존 `#/bookmarks`는 호환용으로 `#/history`로 이동한다.
- 학습 기록은 북마크와 오답 문제를 합쳐 보여주며 기록 유형, 과목, 출처 필터와 모두 펼치기/모두 접기를 제공한다.
- 선택 옵션은 `sessionStorage`에 저장한다: `pt.selectedSources`, `pt.practiceScope`, `pt.practiceOptions`, `pt.currentSession`.
- 영구 사용자 데이터는 `localStorage`에 저장한다: `pt.bookmarks`, `pt.wrongAnswers`.
- 데이터 관리의 내보내기 파일명은 `passture-user-data-YYYYMMDD-HHMMSS.json` 형식이다.
- 개발 서버에서 `public/data` JSON이 없는 경우 `src/lib/data-loader.ts`가 dev 모드에 한해 원본 YAML을 직접 읽어 fallback한다. 배포 빌드는 기존처럼 `pnpm data:build` 산출 JSON을 사용한다.
- GitHub Pages Source는 `GitHub Actions` + `Static HTML` 선택이 맞고, `https://passture.logonme.click/data/catalog.json` 정상 응답 확인 완료.
