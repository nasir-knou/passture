방통대 기말고사 대비 사이트를 만들고자해

크게 사이트관리와 문제관리 2개 측면의 요구사항이 있어

# 사이트관리

- 비용 절감을 위해 github pages의 정적페이지 사용(서버 운영/저장 비용 0)
- 과목별, 문제유형별 저장된 문제를 풀 수 있는 플랫폼으로 제공
- 틀렸던 문제를 북마크해두고 다시 볼 수 있는 기능 필요
- 현재 테스트할 과목은 다음과 같지만 추후 다른 과목도 추가할 수 있는 확장성있는 아키텍쳐 필요
  - 운영체제
  - 이산수학
  - 알고리즘
  - Java프로그래밍
- 문제의 유형(출처)는 다음과 같음
  - 기출3개년(17~19) (연도별 25문제)
  - 교재워크북 (챕터별 문제 수 다양)
  - 강의연습문제 (강의별 문제 수 다양)
- 각 과목별 유형을 선택하는 디렉토리 구조나 과목별로 체크리스트로 풀 문제 골라오는 방식등 고민중
- 선택된 문제에서 문제순서 변경, 선택지 순서 변경등의 익숙함 제거 필요
- 사용자는 문제를 보고 선택지를 체크하고 확인을 누르면 정답이 표시되고 해설이 표시됨. 해설은 문제에 같이 저장되어있음

# 문제관리

- 문제는 별도의 파일로 저장하고 사이트에서 필요시 불러오는 형식. JSON 또는 YAML 생각 중이지만 적절한 저장형식 추천
- 위의 문제유형 및 출처별로 단일파일로 관리할 예정
- 단일파일내에서 문제마다 번호의 개별 식별자 필요
  - 기출의 경우 17-01, 19-23등 yy-번호
  - 교재의 경우 b01-03, b07-08등 b챕터-번호
  - 강의의 경우 l01-03, l07-08등 l강의-번호
- 문제는 각자하나의 문제가 되는 경우도 있지만, 주어진 코드를 보고 다음 3개의 문제를 푸시오 처럼 공통지문을 여러 문제에서 참고해야하는 경우있음
  - 공통 부분을 분리하고 삽입할지 아니면 개별 문제에 모든 요소를 완벽히 복제해서 넣을지
- 문제에는 이미지 첨부가 필요한 경우도 있음. 이런 경우 github repo에 이미지파일을 그대로 저장하고 불러올 예정
- 문제는 O/X 또는 4지선다의 방식이지만 문제 오류로 인해 중복정답인 경우도 있음.
  - 정답이 1, 2, 3, 4 일수도있고 A:1,2 B:1,3 C:1,4, .. F:1,2,3 이런식으로 알파벳으로 표시하여 중복정답 대응할수도있음
